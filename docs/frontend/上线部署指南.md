### 步骤3: 部署到服务器

#### 方式1: 直接部署（Nginx）

```bash
# 1. 复制构建产物到服务器
scp -r dist/* user@server:/var/www/heikeji-web/

# 2. 配置Nginx
sudo vim /etc/nginx/sites-available/heikeji-web
```

**Nginx配置示例**:

```nginx
server {
    listen 80;
    server_name www.heikeji.com heikeji.com;
    
    # 重定向到HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name www.heikeji.com heikeji.com;
    
    # SSL证书配置
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    # 前端文件根目录
    root /var/www/heikeji-web;
    index index.html;
    
    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/javascript application/xml+rss application/json;
    
    # 缓存静态资源
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # 前端路由支持（单页应用）
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API代理
    location /api {
        proxy_pass https://api.heikeji.com;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # 错误页面
    error_page 404 /index.html;
    error_page 500 502 503 504 /50x.html;
    location = /50x.html {
        root /var/www/heikeji-web;
    }
}
```

#### 方式2: Docker部署

**Dockerfile**:

```dockerfile
# 构建阶段
FROM node:18-alpine AS builder

WORKDIR /app

# 复制依赖文件
COPY package*.json ./
RUN npm install --legacy-peer-deps

# 复制源代码
COPY . .

# 构建
RUN npm run build

# 运行阶段
FROM nginx:alpine

# 复制构建产物
COPY --from=builder /app/dist /usr/share/nginx/html

# 复制Nginx配置
COPY nginx.conf /etc/nginx/conf.d/default.conf

# 暴露端口
EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

**构建和运行**:

```bash
# 1. 构建Docker镜像
docker build -t heikeji-web:latest .

# 2. 运行容器
docker run -d \
  --name heikeji-web \
  -p 80:80 \
  -p 443:443 \
  -v /path/to/ssl:/etc/nginx/ssl \
  heikeji-web:latest

# 3. 查看运行状态
docker ps
docker logs heikeji-web
```

## 🔒 安全配置

### 1. HTTPS配置

```nginx
# 强制HTTPS
server {
    listen 80;
    server_name www.heikeji.com;
    return 301 https://$server_name$request_uri;
}

# SSL配置
ssl_protocols TLSv1.2 TLSv1.3;
ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
ssl_prefer_server_ciphers off;
ssl_session_cache shared:SSL:10m;
ssl_session_timeout 1d;
```

### 2. 安全头部

```nginx
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self'; connect-src 'self' https://api.heikeji.com;" always;
```

### 3. 敏感信息保护

- [ ] API密钥不提交到代码仓库
- [ ] 使用环境变量管理敏感配置
- [ ] 生产环境禁用调试模式

## 🔄 回滚方案

### 快速回滚

```bash
# 1. 备份当前版本
cp -r /var/www/heikeji-web /var/www/heikeji-web-backup-$(date +%Y%m%d-%H%M%S)

# 2. 回滚到上一版本
cp -r /var/www/heikeji-web-backup-previous /var/www/heikeji-web

# 3. 重启Nginx
sudo nginx -s reload

# 4. 验证回滚
curl -I https://www.heikeji.com
```

### Docker回滚

```bash
# 1. 停止当前容器
docker stop heikeji-web
docker rm heikeji-web

# 2. 运行旧版本容器
docker run -d \
  --name heikeji-web \
  -p 80:80 \
  -p 443:443 \
  heikeji-web:previous-tag

# 3. 验证回滚
docker ps
curl -I https://www.heikeji.com
```

```javascript
// 前端性能监控
// 在main.ts中添加
if (import.meta.env.PROD) {
```

```bash

cat > /usr/local/bin/health-check.sh << 'EOF'
#!/bin/bash

    exit 0
```

## 🆘 常见问题

### 问题1: 刷新页面404

**原因**: 单页应用路由问题

**解决**: 配置Nginx try_files

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

### 问题2: API请求跨域

**原因**: CORS配置问题

**解决**: 检查后端CORS配置或Nginx代理配置

### 问题3: 静态资源加载失败

**原因**: 路径问题或缓存问题

**解决**: 
- 检查资源路径
- 清除CDN缓存
- 检查文件权限

### 问题4: 构建产物过大

**解决**: 
- 启用代码分割
- 压缩资源
- 使用CDN
