## 3. API 请求缓存

### 缓存策略
已在 `src/utils/request.js` 中实现：

```javascript
// 缓存配置
cache: {
  enabled: true,           // 启用缓存
  defaultCacheTime: 5 * 60 * 1000,  // 默认5分钟
}
```

### 缓存规则
- 仅 GET 请求会缓存
- 可通过 `config.cache = false` 禁用单个请求
- 可通过 `config.cacheTime` 自定义缓存时间

### 清除缓存
```javascript
import { clearRequestCache } from '@/utils/request'

// 清除所有请求缓存
clearRequestCache()
```

## 4. Vite 构建优化

### 当前配置 (vite.config.js)

```javascript
build: {
  outDir: 'dist',
  sourcemap: false,           // 禁用 sourcemap
  minify: 'esbuild',          // 使用 esbuild 压缩
  rollupOptions: {
    manualChunks: {
      vue: ['vue', 'vue-router', 'pinia'],
      element: ['element-plus'],
      echarts: ['echarts'],
      vendor: ['axios', 'js-cookie', 'nprogress'],
    },
  },
}
```

### 优化效果
- Vue 生态独立 chunk
- Element Plus 独立 chunk
- ECharts 独立 chunk（按需加载）
- 第三方库独立 chunk

## 9. 缓存策略

### 本地缓存
```javascript
import { useLocalCache, useSessionCache } from '@/utils/cache-adapter'

// localStorage
const localCache = useLocalCache()

// sessionStorage
const sessionCache = useSessionCache()
```

### Redis 缓存
```javascript
import { defaultCacheManager } from '@/utils/cache'

// 设置缓存
await defaultCacheManager.set('key', data, ttl, ['tag'])

// 获取缓存
const data = await defaultCacheManager.get('key')

// 缓存预热
await defaultCacheManager.warmup([
  { key: 'product:list', value: [...] }
])
```

## 10. 性能优化建议

### 生产环境
1. 开启 CDN 加速
2. 配置 HTTP/2
3. 开启浏览器缓存
4. 使用 Service Worker 离线缓存

### 开发环境
1. 使用 Vite HMR
2. 启用依赖预加载
3. 使用 fastRefresh

### 监控指标
| 指标 | 目标值 |
|------|--------|
| First Contentful Paint | < 1.5s |
| Largest Contentful Paint | < 2.5s |
| Time to Interactive | < 3.5s |
| Cumulative Layout Shift | < 0.1 |
| API 响应时间 | < 500ms |
| 缓存命中率 | > 80% |
