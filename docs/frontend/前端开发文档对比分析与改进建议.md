## 四、改进建议（按优先级排序）

### 🔴 P0 — 必须立即处理（影响VI合规性和核心功能完整性）

#### 建议 1：统一科大蓝标准色值

**问题描述：**
当前 Tailwind 配置中主色调为 `#003B80`，与文档规定的 VI 标准色值 `#000AB0`（CMYK: C=100, M=95, Y=25, K=0）不一致。

**涉及文件：**
- `heikeji-web/tailwind.config.js` — 第 12 行 `primary.DEFAULT`
- `heikeji-web/src/tokens/colors.ts` — lightColorTokens 中的 primary 定义
- 可能涉及的硬编码颜色引用

**修改方案：**

```js
// tailwind.config.js — colors.primary 修改
primary: {
  DEFAULT: '#000AB0',    // 科大蓝 VI 标准色 C=100 M=95 Y=25 K=0
  dark: '#000880',       // 深蓝
  light: '#1A5FB4',      // 浅蓝
  50: '#E6ECFA',
  100: '#BDD0F0',
  200: '#8FAAE4',
  300: '#6184D8',
  400: '#335ECC',
  // ...
}
```

同时需同步更新 Element Plus 主题覆盖变量和 CSS 变量。

#### 建议 2：补充校徽组件

**问题描述：**
文档明确要求校徽出现在顶部导航左侧和个人中心头像旁，当前项目中未发现校徽组件。

**实施方案：**

1. 准备校徽资源文件：
   - 获取黑科大校徽 SVG 或高清 PNG（圆形，外环中英文校名，内部书籍+博士帽主楼图案+1947）
   - 放置路径：`heikeji-web/src/assets/logo/usth-logo.svg`

2. 创建校徽组件：
   - 新建 `heikeji-web/src/components/global/USTHLogo.vue`
   - 支持多种尺寸变体（sm/md/lg）、支持点击返回首页

3. 集成位置：
   - 顶部全局导航栏左侧（替换或补充文字 Logo）
   - 个人中心页面用户头像旁
   - 登录页面品牌区域
   - 页脚版权区域

#### 建议 3：新增学工办理模块

**问题描述：**
学工办理是黑科大校园平台的核心差异化功能，对接 USTH 智慧学工系统，目前完全缺失。

**建议路由结构：**

```
/student-affairs                    学工办理（Layout.vue）
  ├── /student-affairs/overview     学工首页（待办事项概览）
  ├── /student-affairs/leave        请假申请与销假
  ├── /student-affairs/aid          助学金/困难生申请
  ├── /student-affairs/military     军训服装预定（尺码选择/订单查询/核销码展示）
  ├── /student-affairs/campus-card  校园卡服务（充值/挂失/查询/制卡指引）
  └── /student-affairs/policy       资助政策与绿色通道
```

**需要新建的文件：**
- `src/views/student-affairs/Layout.vue` — 模块布局
- `src/views/student-affairs/Overview.vue` — 学工首页
- `src/views/student-affairs/Leave.vue` — 请假申请
- `src/views/student-affairs/Aid.vue` — 助学金申请
- `src/views/student-affairs/Military.vue` — 军训服装预定
- `src/views/student-affairs/CampusCard.vue` — 校园卡服务
- `src/views/student-affairs/Policy.vue` — 资助政策
- `src/api/studentAffairs.ts` — API 接口定义
- `src/stores/studentAffairs.ts` — Pinia 状态管理
- `src/types/studentAffairs.ts` — TypeScript 类型定义

**后端对接需求：**
- 对接 USTH 智慧学工系统 API
- 对接学校统一身份认证（SSO）
- 对接校园卡管理系统接口

## 六、总结

### 整体评价

当前 heikeji-web 项目具备以下优势：

1. **技术栈先进**：Vue 3.5 + Vite 8 + TS 5.9 + Tailwind 4 + Pinia 3，均为各生态最新稳定版
2. **工程化成熟**：完整的 Design Tokens 体系、ThemeEngine 主题引擎、Composables 分层架构、安全加固模块、性能监控、E2E 测试、Docker 部署就绪
3. **通用功能完善**：电商购物、外卖配送、二手交易、社区论坛、个人中心等通用校园生活功能已完成
4. **品牌融入良好**：校训展示区、科大蓝配色体系、办学数据展示等体现了学校特色

### 主要差距

项目的主要差距在于**黑科大特色业务功能的覆盖度**：

- 电商/外卖/二手等**通用型校园功能**已经非常完善
- 但**学工办理、缴费中心、AI教学**等学校独有、对接内部系统的**差异化核心功能**尚处于缺失或骨架状态

### 核心建议

> **优先完成 VI 规范对齐（成本低、见效快），然后集中力量补齐学工办理和缴费中心两大核心模块，使平台从"通用校园电商平台"升级为"真正的黑科大一站式校园服务平台"。**
