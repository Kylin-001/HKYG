## 🧪 核心业务流程测试

### 流程1: 用户注册/登录流程

```
1. 用户注册 (POST /user/register)
   - 验证邮箱格式
   - 验证密码强度
   - 检查学号是否已存在

2. 用户登录 (POST /user/login)
   - 验证账号密码
   - 返回Token和用户信息

3. 获取用户信息 (GET /user/info)
   - 使用Token获取用户详情
```

**测试要点**:
- [ ] 注册成功返回正确数据
- [ ] 登录成功返回Token
- [ ] Token能正确获取用户信息
- [ ] Token过期能正确刷新

### 流程2: 商品浏览/下单流程

```
1. 获取商品列表 (GET /products)
2. 获取商品详情 (GET /products/{id})
3. 添加购物车 (POST /cart/items)
4. 创建订单 (POST /orders)
5. 创建支付 (POST /payments)
6. 查询支付状态 (GET /payments/{id})
```

**测试要点**:
- [ ] 商品列表分页正常
- [ ] 商品详情数据完整
- [ ] 购物车添加成功
- [ ] 订单创建成功
- [ ] 支付流程正常

### 流程3: 外卖点餐流程

```
1. 获取商家列表 (GET /takeout/merchants)
2. 获取商家详情 (GET /takeout/merchants/{id})
3. 创建外卖订单 (POST /takeout/orders)
4. 支付订单 (POST /payments)
5. 获取订单状态 (GET /takeout/orders/{id})
```

**测试要点**:
- [ ] 商家列表排序正常
- [ ] 商家详情包含菜品
- [ ] 订单创建计算金额正确
- [ ] 订单状态更新正常
