# 文档整理计划

## 一、整理目标
1. 建立清晰的文档分类结构
2. 减少根目录零散文档
3. 统一文档命名规范
4. 优化文档存放位置
5. 便于团队成员查找和使用

## 二、文档分类结构设计

```
heikeji-mall/
├── README.md                      # 项目主README
├── docs/                          # 主文档目录
│   ├── architecture/              # 架构相关文档
│   ├── development/               # 开发相关文档
│   ├── deployment/                # 部署相关文档
│   ├── operation/                 # 运维相关文档
│   ├── test/                      # 测试相关文档
│   ├── module/                    # 模块说明文档
│   ├── api/                       # API接口文档
│   └── project/                   # 项目管理文档
├── heikeji-admin/                 # 管理后台模块
│   └── README.md                  # 模块README
├── heikeji-app/                   # APP API模块
│   └── README.md                  # 模块README
├── heikeji-frontend/              # 前端模块
│   └── docs/                      # 前端模块文档
├── heikeji-mall-service/          # 后端服务模块
│   └── service-payment/           # 支付服务
│       └── PAYMENT_API_GUIDE.md   # 支付API文档
├── heikeji-miniprogram/           # 小程序模块
│   └── README.md                  # 模块README
├── sql/                           # 数据库脚本
│   └── README.md                  # 数据库说明
└── misc/                          # 杂项文档
    ├── issues/                    # 问题解决方案
    └── temp/                      # 临时文档
```

## 三、具体整理步骤

### 1. 根目录文档处理
- 保留：`README.md`
- 移动到docs/project：`PROJECT_BUSINESS_INTRO.md`
- 移动到docs/deployment：所有DEPLOYMENT_*.md, UBUNTU_DEPLOYMENT_DETAILED.md
- 移动到misc/issues：所有问题解决方案文档（MYSQL_*.md, NACOS_INSTALL_FIX.md, NTP_FIX.md, SSH_TROUBLESHOOTING.md, YUM_UPDATE_FIX.md）
- 移动到docs/database：SQL_Management_Plan.md
- 其他零散文档根据内容分类移动

### 2. docs目录优化
- 将架构相关文档移动到architecture/子目录
- 将开发相关文档移动到development/子目录
- 将部署相关文档移动到deployment/子目录
- 将运维相关文档移动到operation/子目录
- 将测试相关文档移动到test/子目录
- 将模块说明文档移动到module/子目录
- 将API接口文档移动到api/子目录
- 将项目管理文档移动到project/子目录

### 3. 模块文档处理
- 各模块的README.md保持在模块根目录
- heikeji-frontend的文档保持在heikeji-frontend/docs/目录
- heikeji-mall-service的服务文档保持在对应服务目录

### 4. .trae/documents/处理
- 将项目相关计划文档移动到docs/project/
- 将部署相关文档移动到docs/deployment/
- 将技术问题解决方案移动到misc/issues/

## 四、文档命名规范
1. 使用中文命名，清晰表达文档内容
2. 避免重复命名，如已有"部署文档.md"，不再创建类似文档
3. 文档名称应包含核心关键词，便于搜索
4. 版本号使用v1.0, v2.0等格式，不使用中文"更新版"

## 五、注意事项
1. 整理过程中保持文档内容不变，仅调整位置和名称
2. 确保所有文档都能被正确访问
3. 记录文档移动情况，便于后续查找
4. 清理重复或过时的文档
5. 确保README.md指向正确的文档位置
