# 黑龙江科技大学校园服务平台重新开发计划

## 1. 项目概述

基于现有黑科易购项目，结合黑龙江科技大学的校园特色和学生需求，重新开发一个综合性校园服务平台。该平台将整合购物、外卖、快递代取、二手市场、失物招领等功能，为学生提供一站式校园生活解决方案。

## 2. 项目定位

- **目标用户**：黑龙江科技大学全体学生，重点服务大三、大四学生
- **核心价值**：节省时间，提高效率，一站式解决校园生活需求
- **项目Slogan**：你的时间很宝贵，跑腿的事交给我们

## 3. 技术栈升级

### 3.1 后端技术栈
- **框架**：Spring Boot 3.2.x
- **微服务**：Spring Cloud 2023.x
- **服务注册发现**：Nacos 2.3.x
- **API网关**：Spring Cloud Gateway 4.x
- **ORM**：MyBatis Plus 3.5.x
- **安全框架**：Spring Security + JWT
- **数据库**：MySQL 8.3.x
- **缓存**：Redis 7.2.x
- **搜索引擎**：Elasticsearch 8.x
- **消息队列**：RabbitMQ 3.13.x
- **分布式事务**：Seata 2.0.x

### 3.2 前端技术栈
- **管理后台**：Vue 3 + Element Plus
- **移动端**：微信小程序 + UniApp
- **构建工具**：Vite 5.x
- **状态管理**：Pinia 2.x

## 5. 核心功能模块

### 5.1 用户中心
- 微信一键登录，绑定学号实名认证
- 个人信息管理（昵称、头像、收货地址）
- 我的钱包（余额、充值、消费记录）
- 我的订单统一管理

### 5.2 商品商城
- 商品分类浏览（零食饮料、日用百货、文具书籍等）
- 商品搜索与排序
- 购物车与下单
- 商家入驻与管理

### 5.3 校园跑腿
- **取快递**：自动填充菜鸟驿站地址，输入取件码即可
- **代购**：指定商品和购买地点
- **代办**：交材料、打印文件等
- 悬赏金设置，跑腿员接单

### 5.4 外卖服务
支持三种配送方式：
1. **外卖柜配送**：选择学校指定的外卖柜，自动分配格口，支持取餐码
2. **特殊地点配送**：外卖员根据实际情况进行配送，支持自定义地点
3. **送到寝室**：直接配送至寝室，需要提供楼栋和房间号

### 5.5 二手市场
- 二手商品发布、浏览、搜索
- 商品分类管理
- 交易流程管理
- 评价体系

### 5.6 失物招领
- 失物/招领信息发布
- 物品分类管理
- 搜索与筛选功能
- 联系机制

### 5.7 校园服务
- 空教室查询
- 校园公告同步
- 校园活动发布与报名

## 6. 数据库设计优化

基于现有数据库设计，进行以下优化：

1. **垂直分库**：按照业务模块划分数据库，提高系统性能和安全性
2. **水平分表**：对大数据量表（如订单表、日志表）进行水平拆分
3. **索引优化**：优化现有索引，添加必要的新索引
4. **数据类型优化**：使用更合适的数据类型，减少存储空间
5. **外键约束优化**：合理使用外键约束，确保数据一致性

## 8. 关键技术点

1. **微服务架构设计**：确保服务之间的低耦合和高内聚
2. **分布式事务处理**：使用Seata解决分布式事务问题
3. **高并发处理**：通过缓存、异步处理、限流等方式提高系统并发能力
4. **数据安全**：实现敏感数据加密、接口鉴权、SQL注入防护等安全措施
5. **用户体验优化**：简化操作流程，提高系统响应速度

## 10. 风险评估与应对措施

1. **技术风险**：采用新技术栈可能遇到兼容性问题
   - 应对措施：提前进行技术调研和验证，选择成熟稳定的技术版本

2. **时间风险**：开发周期可能延长
   - 应对措施：合理规划开发任务，采用敏捷开发方式，定期进行进度检查

3. **数据迁移风险**：现有数据迁移可能出现问题
   - 应对措施：制定详细的数据迁移计划，进行充分的测试，确保数据一致性

4. **用户接受度风险**：新系统可能不被用户接受
   - 应对措施：提前进行用户调研，收集用户需求，不断优化用户体验
