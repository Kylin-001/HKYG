# 黑科易购数据库导入说明

## 快速导入方法

1. **双击运行批处理文件**
   - 找到项目根目录下的 `import_db.bat` 文件
   - 双击运行该文件
   - 按照提示输入 MySQL root 用户的密码
   - 等待导入完成

## 手动导入方法

如果批处理文件导入失败，可以尝试以下手动导入步骤：

### 步骤1：检查MySQL服务
确保MySQL服务已经启动：
- Windows：可以在「服务」中查看 MySQL 服务状态
- 命令行：`net start mysql` (如果未启动)

### 步骤2：登录MySQL
使用命令行工具登录MySQL：
```bash
mysql -u root -p
```
然后输入密码。

### 步骤3：创建数据库
在MySQL命令行中执行：
```sql
CREATE DATABASE IF NOT EXISTS heikeji_mall DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE heikeji_mall;
```

### 步骤4：导入SQL文件
在MySQL命令行中执行：
```sql
source "e:/Program File/黑科易购项目/heikeji-mall/full_db.sql";
```

或者在命令行中直接执行：
```bash
mysql -u root -p heikeji_mall < "e:/Program File/黑科易购项目/heikeji-mall/full_db.sql"
```

## 注意事项

1. **MySQL版本要求**：建议使用MySQL 5.7或更高版本
2. **字符集设置**：确保MySQL默认字符集为utf8mb4
3. **权限问题**：确保MySQL用户有创建数据库和表的权限
4. **路径问题**：执行SQL文件时，确保路径正确
5. **防火墙**：确保MySQL端口（默认3306）未被防火墙阻止

## 常见问题

### 问题1：导入时提示"Access denied for user 'root'@'localhost'"
解决方法：检查MySQL用户名和密码是否正确，确保有足够的权限。

### 问题2：导入时提示"Unknown database 'heikeji_mall'"
解决方法：先手动创建数据库，或者确保SQL文件开头的CREATE DATABASE语句被正确执行。

### 问题3：导入后中文显示乱码
解决方法：确保数据库、表和字段的字符集均为utf8mb4。

### 问题4：导入时提示表已存在
解决方法：可以先删除已存在的数据库，再重新导入：
```sql
DROP DATABASE IF EXISTS heikeji_mall;
```
