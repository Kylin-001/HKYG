**问题**: jsdom与某些ESM模块存在兼容性问题
```
Error: require() of ES Module /home/zky/HKYG/heikeji-frontend/node_modules/@exodus/bytes/encoding-lite.js from /home/zky/HKYG/heikeji-frontend/node_modules/html-encoding-sniffer/lib/html-encoding-sniffer.js not supported.
```

**影响**: 测试运行时出现未处理的错误，但不影响测试执行

**解决方案**:
1. 更新jsdom到最新版本
2. 或者在vitest配置中添加ssr配置
3. 或者忽略这些警告（不影响测试结果）

**问题**: vitest.config.ts在某些环境下存在ESM/CJS兼容性问题

**解决方案**: 将配置文件改为vitest.config.mjs（CommonJS格式）

## 📝 测试最佳实践

### 1. 测试命名规范
- 测试文件：`*.test.ts` 或 `*.spec.ts`
- 测试套件：使用`describe`描述测试组
- 测试用例：使用`it`描述具体测试

### 2. 测试结构
```typescript
describe('Component Name', () => {
  beforeEach(() => {
    // 每个测试前的准备
  })

  afterEach(() => {
    // 每个测试后的清理
  })

  it('should do something', () => {
    // 测试逻辑
  })
})
```

### 3. 断言使用
- 使用`expect`进行断言
- 使用`toBe`、`toEqual`等匹配器
- 使用`toContain`、`toContainEqual`等包含断言

### 4. Mock使用
- 使用`vi.fn()`创建mock函数
- 使用`vi.mock()`mock模块
- 在`beforeEach`中设置mock
- 在`afterEach`中清理mock
